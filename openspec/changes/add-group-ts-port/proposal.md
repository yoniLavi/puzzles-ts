# add-group-ts-port

## Why

Group is a real, complete, fully-playable upstream puzzle that Simon shelved in
`puzzles/unfinished/` only because it is esoteric and *hard* ‚Äî not because it
lacks anything. It is a Latin-square puzzle over a group's Cayley table, so it
reuses the already-ported shared `latin.ts` almost entirely, adding only a thin
group-axiom deduction layer (associativity + identity). Porting it is therefore
cheap relative to its novelty, exercises `latin.ts` as a fifth consumer, and ‚Äî
exactly as `separate` did ‚Äî brings an unfinished-but-complete upstream game into
this fork's deliberately-expanded catalog.

The one genuine product question is whether a puzzle this niche should be
surfaced to players at all (upstream's own stated reason for leaving it
unfinished). This change flags that decision for the owner rather than presuming
it.

## What Changes

- **`src/native/games/group/`** ‚Äî the game, on the established multi-file port
  shape (`state` / `solver` / `generator` / `render` / `index`).
- **Reuse of `src/native/engine/latin.ts`** as the entire Latin-square engine.
  Group contributes only its two difficulty-keyed user-solvers (associativity +
  identity fill at Normal; identity-hidden elimination at Hard) and its
  associativity `valid` callback, wired through the existing `latinSolver`
  config. Extreme = generic set-elimination + forcing, Unreasonable = the shared
  recursion ‚Äî no new solver machinery.
- **The group data table** (`groupdata` / `groups`, 89 groups across orders
  2‚Äì26, generated by upstream's `group.gap`) transcribed verbatim as static TS
  data, with the BFS-decompress-and-shuffle generator over it.
- **Identity mode** (`params.id`) and its element-numbering remap
  (`e,a,b,c,d,f,‚Ä¶` vs `a,b,c,d,e,‚Ä¶`) as an encode/decode + display concern.
- **The two visual aids** upstream ships and this fork should keep: drag-to-
  reorder rows/columns (`sequence`) and the subgroup/coset **dividers**.
- **`findMistakes`** (Group is uniquely solvable, so Check & Save applies) plus
  the pencil-mark note-taking UX, per the playbook.
- **A byte-match differential** against a new `puzzles/auxiliary/group-trace.c`,
  across sizes, both identity modes, and every difficulty.
- **Stage 2, on owner acceptance**: register `group` in the main catalog with
  `TS_PORTED` (no `group.c`/wasm ‚Äî the `separate` precedent), delete
  `puzzles/unfinished/group.c` (and its `unfinished/CMakeLists.txt` entry), and
  rebuild.

Explicitly **not** in this change:

- **An explained hint** ‚Äî its own change, per every prior port. Group is a
  strong Palisade-bar candidate (associativity is a teachable deduction), but no
  hint ships here.
- **The header's TODO solver techniques** (inverse deductions, hard-mode
  associativity over *possible* numbers, element-order inference) ‚Äî upstream
  never implemented them and its five difficulties stand without them. Out of
  scope; noted for a future strengthening change.

## Impact

- Affected specs: new `group` capability; `ts-migration` (the catalog gains an
  expansion-beyond-upstream game, alongside `separate`).
- Affected code: new `src/native/games/group/`, registration in
  `ts-ported-ids.ts` + `games/index.ts`, new `puzzles/auxiliary/group-trace.c`,
  and (stage 2) a `puzzle(group ‚Ä¶ TS_PORTED)` entry in the main
  `puzzles/CMakeLists.txt`.
- No icon work: `src/assets/icons/group-{64,128}d8.png` already exist, so the
  `puzzle-icons` obligation is already met.
- Not on the critical path to `retire-c-engine`: Group's C lives in
  `unfinished/` and is not compiled into the shipped WASM, so deleting it frees
  no runtime but tidies the reference tree.
