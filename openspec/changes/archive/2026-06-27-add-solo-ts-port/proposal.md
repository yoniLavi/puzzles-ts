# Proposal: Port Solo to native TypeScript

**Status**: Proposed

## Why

Solo (the Times's "Sudoku", and the family it spawned) is the last member of the
Latin-square-family migration thread (Unequal → Keen → Undead → Solo) and the
**largest game in the collection** — `solo.c` is ~5790 lines, vs ~2665 for Keen
and ~4500 for Galaxies (the biggest port so far). The player fills a `cr × cr`
grid (`cr = c·r`) so every row, every column, and every sub-block contains each
digit `1..cr` exactly once, with a subset of cells given.

Unlike the rest of the family, **Solo does not ride on the shared
`engine/latin.ts` framework.** Upstream `solo.c` carries a *bespoke* solver — its
own `struct solver_usage` candidate cube and ~14 deductive techniques across two
independent difficulty axes — and its own RNG-driven generator. So this port adds
no reuse to `engine/latin.ts`; it is a self-contained, deep solver/generator port.
It is still the natural "finish the family" milestone and the highest-known-value
remaining game, but the reuse story that justified Keen/Unequal does not apply.

Solo is **four variants in one game**, and they compose:

- **Standard** — rectangular `c × r` sub-blocks.
- **Jigsaw** (`r == 1`) — irregular sub-blocks generated by `divvy_rectangle`
  (`divvy.c`, a Solo-only leaf) instead of the rectangular grid.
- **X** (`xtype`) — the two main diagonals must also contain every digit.
- **Killer** (`killer`) — a second cage partition with digit-sum clues (no
  arithmetic ops, only sums), on top of the Latin + block constraints.

It carries **two difficulty axes**: the standard solver difficulty
(`DIFF_BLOCK/SIMPLE/INTERSECT/SET/EXTREME/RECURSIVE` — Trivial..Unreasonable) and
the killer-cage difficulty (`DIFF_KSINGLE/KMINMAX/KSUMS/KINTERSECT`).

It has **no long-tail-risk entanglements** (playbook §1): no
`midend_supersede_game_desc` (the desc is fixed at generation), no
undo-by-state-equality (every move is a definite cell/pencil toggle; completion is
locally decidable via `check_valid`), and no `#ifdef EDITOR` input. It uniquely
determines its solution, making it a natural `findMistakes` (Check & Save) carrier
and a strong future `hint()` candidate (a separate change).

## What Changes

- **New `src/native/games/solo/` port** implementing
  `Game<SoloParams, SoloState, SoloMove, SoloUi, SoloDrawState, SoloMistake>`:
  - `state.ts` — `SoloParams { c, r, symm, diff, kdiff, xtype, killer }`
    (`cr = c·r`; jigsaw is `r === 1`); the preset list; the param codec
    (`{c}`/`{c}x{r}`/`{c}j` + `x`/`k` flags + `m8/m4/md4/m2/md2/r4/r2/a` symmetry +
    `dt/db/di/da/de/du` difficulty, full vs non-full as upstream); the desc codec
    (givens grid encoding; for jigsaw a trailing block structure; for killer a
    trailing cage block structure + cage-sum grid); `validateParams`/
    `validateDesc`; the immutable `SoloBlocks` (the block partition `Dsf` + helper
    maps) and optional killer-cage structure; mutable `grid`/`pencil`; `cloneState`;
    the move/UI types; the completion check.
  - `solver.ts` — the bespoke `solver_usage` candidate cube and the deductive
    techniques across both axes, ported faithfully (standard: positional/numeric
    elimination, block/row/column intersections, set elimination, the "extreme"
    forcing chains, and the bounded `DIFF_RECURSIVE` guess-and-verify; killer:
    single-cell sums, min/max elimination, sum combinations, cage/line
    intersections), with the X-diagonal and jigsaw-block constraints folded in as
    extra constraint groups. Returns a difficulty sentinel
    (impossible/ambiguous/unfinished or the reached difficulty), used by the
    generator (uniqueness + grading), `solve()`, and `findMistakes`.
  - `generator.ts` — `newDesc`: generate a full solution grid (filling under all
    active constraints, jigsaw blocks via the lazily-ported `divvy`, killer cages
    via the cage divvy + sum assignment), then remove givens down to the symmetry
    orbit / target difficulty by re-running the graded solver (the solver-gated
    minimiser, playbook §4.4). Symmetry handling for all `SYMM_*` modes. Aim for a
    byte-match differential where the RNG path allows; fall back to verdict-record
    where it does not (decided in `design.md`, playbook §4.3/§4.8).
  - `render.ts` — palette index-for-index with the C enum; the grid with sub-block
    borders (rectangular *and* jigsaw-irregular via the block `Dsf`); killer cage
    outlines (dashed insets) + cage-sum labels; X-diagonal shading; givens vs
    player digits; auto-sized pencil-mark grids; cursor + pencil highlight; the
    fork's pencil-mode corner indicator; the Check & Save mistake overlay; the
    completion flash; an `Int32Array` per-tile diff cache with every overlay in the
    diff key (playbook §3.2).
  - `index.ts` — `Game` glue: `interpretMove` (cell select; left = real entry,
    right = pencil with the sticky-pencil + filled-cell rules; digit/backspace/
    space honouring pencil mode + no-op suppression + auto-pencil; `M` → mark-all;
    keyboard cursor), `executeMove`, `status`, `solve` (re-derive; `aux` if needed),
    `findMistakes`, the full pencil-mark UX (`canMarkAll`, sticky-pencil +
    auto-pencil + keep-highlight prefs), `describeParams` (variant-aware keys for
    the config-summary header, playbook §3.4), `colours`, `registerGame`.
- **`divvy` ported lazily as a Solo-local leaf** (`src/native/games/solo/divvy.ts`
  or, if a second consumer ever appears, promoted to `engine/`). It is Solo-only
  upstream (`grep` confirms only `solo.c` includes it), so it is deleted with
  `solo.c` at acceptance.
- **`findMistakes` (Check & Save).** Solo re-solves from the givens (and killer
  cages) to the unique solution and flags every player cell that contradicts it,
  plus the note-mistake convention (an empty cell whose non-empty pencil notes have
  crossed out its solution digit), so the shipped Check & Save control hard-blocks
  a wrong board.
- **Differential.** A gated differential (`solo-differential.test.ts` vs a frozen
  C trace) across the variants and difficulties. Byte-match where the generator is
  RNG-faithful and deterministic to the desc; verdict-record (decode C boards, run
  the TS solver, assert the recorded difficulty / unique-solvability) where a
  `qsort`-style or otherwise non-portable ordering makes byte-match infeasible. The
  call is made per-variant in `design.md` after reading the generator.
- **Stage-1 registration only.** Add `solo` to `ts-ported-ids.ts` and import it in
  `games/index.ts` so the TS impl serves it for owner smoke-testing across all four
  variants. The `TS_PORTED` flag + `puzzles/solo.c` + `puzzles/divvy.c` deletion
  happen **only on owner acceptance**, per the two-stage parity gate.

## Impact

- **Affected specs:** new `solo` capability (ADDED requirements: Game interface +
  variants, param codec, desc codec, generator, the bespoke graded solver, moves +
  cursor, rendering, the pencil-mark UX + preferences, mistake-checking).
- **Affected code:** new `src/native/games/solo/*` (incl. local `divvy.ts`); one
  line each in `ts-ported-ids.ts` and `games/index.ts`;
  `puzzles/auxiliary/solo-trace.c` + its `cliprogram()` line (deleted with
  `solo.c` at acceptance). No change to `solo.c`/`divvy.c` until owner acceptance
  (stage 2). No change to `engine/latin.ts` (Solo does not use it).

## Out of scope

- An explained `hint()` is a **separate** change (`add-solo-hint`), to be done in
  a new session once the base port is accepted. Solo's graded deductive solver
  makes it a strong Palisade-bar candidate, but per the hint-authoring guide that
  is its own parity-gated change.
- Printing (`game_print`) — no TS replacement fork-wide; out of scope for every
  port so far.
- Grid sizes upstream rejects (`validate_params`) — the port matches upstream's
  bounds (e.g. killer dimensions `< 10`), not new sizes.
